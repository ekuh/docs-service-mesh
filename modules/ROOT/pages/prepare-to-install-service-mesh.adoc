[[software-requirements]]
== Software Requirements

Before you download and install Anypoint Service Mesh, make sure you have:

* Installed the following applications:
** Kubernetes 1.12+ or a newer cluster, with RBAC (Role-Based Access Control) enabled
** Helm 2.13.0 or later
*Tip:* To verify installed version, use the `helm version`. 
For more information about how to install Helm, see the Helm installation documentation. .
** Kubectl version 1.12 or later (compatible with the cluster) is installed on your local machine
*Tip:* To verify installed version, use the `kubectl version`.
** One of the following environments on which to run Kubernetes:
*** Google Kubernetes Engine (GKE)
*** Amazon EKS
*** Azure Kubernetes Service (AKS)
* Not installed Istio in the cluster
* Optionally, If you want to enable automatic sidecar injection, you must:
** Install Kubernetes 1.12+ with admissionregistration API 
*Tip:* To verify the installation, use the command `kubectl api-versions | grep admissionregistration.k8s.io/v1beta1`.
** Configure kube-apiserver process with the admission-control flag to “MutatingAdmissionWebhook”
For more information about how to turn on, see tke Kubernetes documentation.
** Add and list ValidatingAdmissionWebhook admission controllers in the correct order


[[licensing-requirements]]
== Licensing Requirements

You require a Anypoint Service Mesh license to perform the installation. If you do not have a license, contact your MuleSoft representative.


[[permissions-and-roles-requirements]]
== Permissions and Roles Requirements

Before you begin to download and install Anypoint Service Mesh, you must have the following roles and permissions:
 
* API Manager Environment Administrator permission or Organization Administrator role for your organization to access the client ID and client secret credentials.
** Obtain the client ID and client Secret credentials from Access Management. You will require these credentials of your organization or environment to connect Service Mesh to Anypoint Platform. MuleSoft recommends that you use the environment credentials instead of the organization credentials.
For more information about how to obtain the credentials, see API Manager documentation.
* Cluster-admin role access to the Kubernetes cluster
