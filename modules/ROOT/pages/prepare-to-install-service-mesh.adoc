To start using Anypoint Service Mesh, you must have installed specific applications and must have specific roles and permissions. This section describes the following requirements:

* Software Requirements (will link to appropriate sections)
* Licensing Requirements
* Permissions and Roles Requirements

== Software Requirements

Before you download and install Anypoint Service Mesh, make sure you have:

* Installed the following applications:
** Kubernetes 1.12+ or a newer cluster, with RBAC (Role-Based Access Control) enabled
Helm 2.13.0 or later
Tip: To verify installed version, use the command: > helm version. 
For more information about how to install Helm, see the Helm installation documentation. https://helm.sh/docs/install/.
** Kubectl version 1.12 or later (compatible with the cluster) is installed on your local machine:
For more information about how to install Kubectl, see https://kubernetes.io/docs/tasks/tools/install-kubectl/#before-you-begin 
* Not installed Istio in the cluster
* Optionally, If you want to enable automatic sidecar injection, you must:
** Install Kubernetes 1.12+ with admissionregistration API 
Tip: To verify the installation, use the command kubectl api-versions | grep admissionregistration.k8s.io/v1beta1
** Configure kube-apiserver process with the admission-control flag to “MutatingAdmissionWebhook”
For more information about how to turn on, see https://kubernetes.io/docs/reference/access-authn-authz/admission-controllers/#how-do-i-turn-on-an-admission-controller
For more information about kube-apiserver, see https://kubernetes.io/docs/reference/access-authn-authz/admission-controllers/#how-do-i-turn-on-an-admission-controller
** Add and list ValidatingAdmissionWebhook admission controllers in the correct order



== Licensing Requirements
Before you begin to download and install Anypoint Service Mesh, email the Service Mesh team at servicemesh@mulesoft.com for the required licenses for Beta.


== Permissions and Roles Requirements
Before you begin to download and install Anypoint Service Mesh, you must have the following roles and permissions:
 
* Environment Administrator or Organization Administrator for your organization to access the client ID and client secret credentials.
* Cluster-admin role access to the Kubernetes cluster
* Obtain the Client ID and Client Secret credentials from Access Management.
For more information about how to obtain the credentials, see https://docs.mulesoft.com/api-manager/2.x/org-credentials-config-mule4
* Obtain the API ID of the application that you want to bind to the Service Mesh adapter, after your provision it. 
For more information about how to obtain API ID, see https://docs.mulesoft.com/api-manager/2.x/create-instance-task
